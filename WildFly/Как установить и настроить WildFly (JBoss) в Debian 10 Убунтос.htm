Как установить и настроить WildFly (JBoss) в Debian 10

28/12/2019 Админ Ubuntu Комментарии

WildFly, ранее известный как JBoss, - это бесплатный кроссплатформенный сервер приложений с открытым исходным кодом, который в настоящее время разрабатывается Red Hat. WildFly написан на Java и помогает вам в создании великолепных приложений. Благодаря подключаемым подсистемам вы можете настроить приложение так, как вам нужно. В этой статье мы расскажем, как администратор Debian может установить и настроить Wildfly в своей системе.

Мы запустили команды и процедуры, упомянутые в этой статье, в системе Debian 10 Buster.

Установить WildFly

Администратор Debian / (пользователь sudo) может установить стабильный сервер приложений WildFly, тщательно выполнив следующие действия:

Шаг 1: Откройте Терминал или подключитесь к серверу по SSH

Мы будем использовать Debian Командная строка, Терминал, для того, чтобы установить и настроить WildFly. Вы можете открыть приложение «Терминал» с помощью поиска в Панели запуска приложений следующим образом:

Шаг 2. Обновление индекса репозитория

Чтобы установить последнюю доступную версию программного обеспечения из интернет-репозиториев, ваш локальный индекс репозитория должен соответствовать их. Для обновления индекса локального репозитория выполните следующую команду от имени sudo: 

$ sudo apt-get update

Обратите внимание, что только авторизованный пользователь может добавлять, удалять и настраивать программное обеспечение в Debian.

Шаг 3: Установите пакет OpenJDK из APT

Пожалуйста, выполните следующую команду от имени sudo, чтобы установить OpenJDK из репозиториев APT: 

$ sudo apt-get install default-jdk

Система может попросить вас ввести пароль для sudo, а также предоставить вам опция Y / n для продолжения установки. Введите Y, а затем нажмите Enter.

Шаг 4: Создайте пользователя и группу для WildFly

В качестве предварительного шага вам необходимо создать пользователя и группу, которые впоследствии будут использоваться для запуска службы WildFly.

Запустите следующую команду для создания новой группы: 

$ sudo groupadd -r wildfly

Запустите следующую команду, которая будет авторизована для запуска службы, в / opt / wildfly 

$ sudo useradd -r -g wildfly -d /opt/wildfly -s /sbin/nologin wildfly

Совет: Вам нужно будет использовать несколько длинных команд при установке WildFly. Вместо того, чтобы вводить команду, вы можете скопировать ее отсюда и вставить в Терминал, используя Ctrl + Shift + V, или используя опцию Вставить в контекстном меню.

Шаг 5: Загрузите установочный файл Wildfly

Теперь мы будем загружать tar.gz для WildFly с официального сайта JBoss.

Во-первых, давайте создадим переменную для определения номера версии WildFly, которую мы хотим загрузить. 

$ Version_Number=16.0.0.Final

Я хочу скачать версию 16.0.0.Final на моей системе. Эта переменная может использоваться во всех командах, где вам нужно указать версию WildFly.

Запустите следующую команду, чтобы загрузить файл tar.gz в папку / tmp вашего Debian. 

$ wget https://download.jboss.org/wildfly/$Version_Number/wildfly-$Version_Number.tar.gz -P /tmp

Шаг 6: Извлеките файл WildFly tar.gz в папку / opt

Программное обеспечение WildFly будет установлено после извлечения загруженного файла .tar.gz. Выполните следующую команду от имени sudo, чтобы извлечь файл в папку / opt.

$ sudo tar xf /tmp/wildfly-$Version_Number.tar.gz -C /opt/

Шаг 7. Создайте символическую ссылку, указывающую на каталог установки WildFly

Выполните следующую команду, чтобы создать символическую ссылку под именем wildfly. Эта ссылка будет указывать на каталог установки Wildfly. 

$ sudo ln -s /opt/wildfly-$Version_Number /opt/wildfly

Шаг 8: Предоставьте доступ группе и пользователю wildfly

Пользователю и группе WildFly необходимо владеть каталогом установки WildFly, чтобы они могли обращаться к WildFly и запускать его.

Для этого выполните следующую команду: 

$ sudo chown -RH wildfly: /opt/wildfly

Шаг 9: Настройте Wildfly для запуска в качестве службы

Пожалуйста, внимательно следуйте этим шагам, чтобы настроить WildFly для запуска в качестве службы Systemd:

1. Создайте каталог, в который мы будем копировать файл wildfly.conf. Этот файл является частью пакета WildFly, который вы скачали и установили. 

$ sudo mkdir -p /etc/wildfly

2. Скопируйте файл wildfly.conf из файлов пакета во вновь созданный каталог с помощью следующей команды: 

$ sudo cp /opt/wildfly/docs/contrib/scripts/systemd/wildfly.conf /etc/wildfly/

3. Откройте файл конфигурации в редакторе Nano с помощью следующей команды: 

$ sudo nano /etc/wildfly/wildfly.conf

Конечно, вы можете использовать любой из ваших любимых текстовых редакторов.

Вот так выглядит файл:

Этот файл пока содержит базовую конфигурацию для автономной системы. Позже мы объясним, как редактировать этот файл для настраиваемых конфигураций, например, при попытке удаленного доступа к административной консоли.

4. Затем скопируйте скрипт launch.sh из пакета WildFly в папку / opt / wildfly / bin /: 

$ sudo cp /opt/wildfly/docs/contrib/scripts/systemd/launch.sh /opt/wildfly/bin/

Кроме того, сделайте скрипт исполняемым с помощью следующей команды: 

$ sudo sh -c 'chmod +x /opt/wildfly/bin/*.sh'

5. Последним копируемым файлом является файл модуля wildfly.service в папку системных служб / etc / systemd / system 

$ sudo cp /opt/wildfly/docs/contrib/scripts/systemd/wildfly.service /etc/systemd/system/

6. Наконец, вы должны сообщить своей системе, что вы добавили новый файл модуля. Это можно сделать, перезагрузив демон systemctl: 

$ sudo systemctl daemon-reload

Теперь вы готовы запустить службу WildFly от имени администратора.

Шаг 10: Запустите сервис WildFly

Теперь сервис WildFly можно запускать как любой другой сервис, распознаваемый Debian.

Запустите следующую команду от имени sudo для запуска службы: 

$ sudo systemctl start wildfly

Вы можете проверить, все ли работает хорошо, проверив состояние службы следующим образом: 

$ sudo systemctl status wildfly

Активное состояние, как указано выше , проверяет, что служба запущена и работает.

Еще одна вещь, которая поможет вам, - это включить службу wildfly при загрузке: 

$ sudo systemctl enable wildfly

Настроить Wildfly

Теперь, когда мы установили WildFly и запустили его как службу, пришло время выполнить некоторые настройки. К ним, в основном, относятся:

  • Настройка брандмауэра
  • Создание защищенного администратора WildFly
  • Проверка успешной настройки
  • Доступ к административной консоли WildFly локально и удаленно

Пожалуйста, выполните следующие шаги один за другим, чтобы использовать WildFly портал надежно.

Шаг 1. Разрешить трафик через порт 8080

Если вы хотите, чтобы к вашему серверу WildFly можно было получить удаленный доступ, вам нужно разрешить трафик через порт 8080.Если на вашем Debian по умолчанию не установлено приложение брандмауэра UFW, выполните следующую команду, чтобы установить его в своей системе: 

$ sudo apt-get install ufw

Для этого выполните следующую команду: 

$ sudo ufw allow 8080/tcp

Шаг 2: Создайте WildFly Administrator

На этом шаге мы будем создавать и настраивать пользователя WildFly. Этот пользователь будет управляющим пользователем / администратором для административной веб-консоли и интерфейса командной строки, которые можно использовать удаленно.

Запустите следующую команду, чтобы запустить скрипт add-user из каталога WildFly: 

$ sudo /opt/wildfly/bin/add-user.sh

Процедура, приведенная ниже, не требует пояснений, но, тем не менее, мы вам поможем.

Затем скрипт предложит вам добавить информацию о новом пользователе. Это включает добавление имени пользователя нового администратора и указание, а затем повторный ввод пароля следующим образом:

Шаг 3: Проверка успешной настройки WindFly

Теперь давайте проверим, работает ли наш сервер WindFly. Откройте браузер и введите следующий URL:

http: // : 8080

Я проверяю настройку на своем локальном хосте:

http: // localhost: 8080 /

Если вы видите что-то похожее на показано выше, мы можем заверить вас, что ваш экземпляр WildFly запущен и работает.

Удаленное управление административной консолью

Чтобы получить удаленный доступ к административной консоли, вам необходимо выполнить небольшие настройки для трех файлов WildFly.

1. Откройте файл wildfly.conf с помощью следующей команды: 

$ sudo nano /etc/wildfly/wildfly.conf

Добавьте следующие строки в конец файла: 

# The address console to bind to

WILDFLY_CONSOLE_BIND=0.0.0.0

Вот так должен выглядеть файл:

Quit файл через Ctrl + X и затем сохраните изменения, нажав y и затем Enter.

2. Откройте файл сценария запуска .sh с помощью следующей команды: 

$ sudo nano /opt/wildfly/bin/launch.sh

Измените выделенные строки на следующие: 

$WILDFLY_HOME/bin/domain.sh -c  -b  -bmanagement 

else

$WILDFLY_HOME/bin/standalone.sh -c  -b  -bmanagement

Выйдите из файла с помощью Ctrl + X и сохраните изменения, нажав y и затем Enter.

После этого перезапустите сервис WildFly с помощью следующей команды: 

$ sudo systemctl restart wildfly

3. Наконец, отредактируйте wildfly.служебный файл с помощью следующей команды: 

$ sudo nano /etc/systemd/system/wildfly.service

Замените выделенную строку следующим: 

ExecStart=/opt/wildfly/bin/launch.sh $WILDFLY_MODE $WILDFLY_CONFIG $WILDFLY_BIND $WILDFLY_CONSOLE_BIND

Поскольку мы изменили файл служебного блока, уведомим систему с помощью следующей команды: 

$ sudo systemctl daemon-reload

Кроме того, перезапустите службу WildFly с помощью следующей команды: 

$ sudo systemctl restart wildfly

Теперь вы можете получить доступ к консоли, введя следующую консоль на удаленном компьютере:

http: // : 9990 / console

Убедитесь, что порт 9990 открыт для трафика на вашем брандмауэре.

Как открыть консоль администрирования CLI

Откройте терминал Debian и перейдите в папку / opt / wildfly / bin, из которой мы будем запускать скрипт CLI: 

$ cd /opt/wildfly/bin/

Затем введите следующую команду для запуска сценарий CL консоли администратора WildFly: 

$ ./jboss-cli.sh --connect
